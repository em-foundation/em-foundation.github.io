    Getting started with EM

    Using the EM language and its runtime requires a cross-development environment, in which hosted tooling will build / load executable EM programs onto target hardware. For the host, you'll use a PC running Windows, Linux, or macOS ;  for the target, you can choose any MCU board for which an em$distro package already exists.

    The EM SDK (described next) comprises a set of software development artifacts which you will download and install onto your host computer. Before doing so, however, ensure that recent versions of Node.js and VS Code reside on your PC — and then install / upgrade these software engines as appropriate.

    At a minimum, the EM SDK requires version 16.3.0 of Node.js and version 1.80.0 of VS Code.  Execute the commands node --version and code --version from your PC's shell to verify this requirement.

    Do not proceed forward if either of these checks fail !!!

    Software development kit

    Go ahead and download the latest version of the EM SDK tailored for your PC's operating system. Delivered as a .zip file, just unzip this file into the $HOME or %HOMEPATH% folder on your PC — which will then populate a sub-folder named .em-sdk containing a variety of artifacts. Going forward, we'll refer to the $HOME/.em-sdk or %HOMEPATH%\.em-sdk folders using the symbol EM-SDK.

    Under the EM-SDK folder you'll find a sub-folder named tools, which contains C / C++ compilers as well as emulation utilities — all used internally by EM to build / load executable target programs. Already located in a well-known place on your PC, EM requires no further setup of these tools.

    The EM-SDK folder contains the EM Builder VS Code extension — found in a file named em-builder-vers.vsix, where vers encodes the extension's major / minor / micro version number followed by a timestamp. EM-SDK also contains a VS Code profile used in conjunction with the EM Builder extension — found in a file named EM.code-profile.

    Target MCU hardware

    The EM SDK — which you've just installed — contains all of the tools required to build / load EM programs targeting any of these boards; you won't need to download any additional, vendor-specific software.  While we encourage you to purchase one (or more) of these boards, you can still learn a great deal about EM without having any target hardware.

    The Texas Instruments CC2340R5 wireless MCU features an Arm Cortex-M0+ CPU together with a familiar suite of peripherals — including a generic 2.4 GHz radio with BLE 5.x support. Texas Instruments also offers an inexpensive LP-EM-CC2340R5 evaluation board in their familiar LaunchPad format — available from TI as well as their distributors.

    You should also purchase this emulator board from TI — unless you already own a "classic" TI LaunchPad with on-board XDS110 support. In that case, you can easily connect this legacy LP to your new LP-EM-CC2340R5 board using a cable supplied by TI.

    EM Builder extension

    At this stage, we can launch VS Code and install its EM Builder extension. To begin, first create an empty folder anywhere on your PC (outside of EM-SDK); this folder will serve as your initial VS Code workspace.
